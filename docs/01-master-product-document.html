1Product Overview & Vision

Mission Clear Build USA gives US construction contractors a single, professional platform to manage their entire business - from first lead to final payment - replacing scattered spreadsheets, paper invoices, and disconnected apps.

What is Clear Build USA?

Clear Build USA is a cloud-based construction business management SaaS built for US-based contractors, general contractors, remodelers, and specialty trades. It covers the complete business lifecycle:

Lead Capture → Quoting → Project Management → Invoicing → Payment Collection

The platform eliminates the need for separate tools like QuickBooks (invoicing), Calendly (scheduling), Slack (communications), and paper/Excel (quotes). Everything lives in one place, accessible on any device.

Core Value Propositions

ValueDescription
Professional imageBranded quotes and invoices with online approval - contractors look like established businesses, not sole proprietors with handwritten notes.
Faster paymentOnline payment via Stripe, automated invoice reminders, and clear payment tracking reduce days-sales-outstanding.
Less admin timeAuto-numbered documents, change orders linked to projects, and one-click quote-to-invoice conversion cut paperwork by hours per week.
Team coordinationAssign crews to projects, share project visibility with clients, and manage roles so everyone sees only what they need.
Client communicationUnified inbox for Email, SMS, and WhatsApp communications. Customers can approve quotes online - no back-and-forth phone calls.
Business intelligenceDashboard with revenue trends, outstanding amounts, and pending quotes gives owners an instant business health check.

Product Vision (3-Year)

Become the #1 all-in-one business platform for US construction contractors with 1–50 employees. Expand from core job management into:

  • Material procurement and supplier management
  • Payroll and labor hour tracking
  • AI-driven estimate generation from job photos
  • Customer-facing project portal with live progress updates
  • Integration marketplace (QuickBooks, Xero, DocuSign, Procore)

Billing Flow

  1. User clicks "Start Monthly Plan" or "Start Yearly Plan" on /subscription
  2. App calls POST /api/billing/checkout → creates Stripe Checkout Session
  3. User redirected to Stripe-hosted checkout page
  4. On success, Stripe sends webhook to POST /api/webhooks/stripe
  5. Webhook updates subscriptions table with Stripe subscription ID, status, and plan
  6. User redirected to /subscription?success=1 with confirmation toast

Trial Expiry Behaviour

  • After 15 days, restricted_mode = true is set on the business
  • Restricted mode shows upgrade prompts on create flows
  • Read-only access to existing data remains available during restriction

7.2 Application Architecture

Pattern: Client Pages + API Routes All app/(app)/ pages are "use client" components that call their corresponding app/api/ routes. The app/(app)/layout.tsx is the only server component - it reads the session and injects user, businesses, and currentBusiness into the sidebar. This keeps auth server-side while keeping UI interactive.

Authentication Flow

  1. User logs in → POST /api/auth/login verifies password, signs JWT with jose
  2. JWT stored as cb_session cookie (HTTP-only, SameSite=Lax)
  3. JWT payload: { id, name, email, businessId, role }
  4. Every API route calls requireSession() from lib/auth.ts
  5. middleware.ts runs before every page render to guard protected routes
  6. Unauthenticated → redirect to /login; no businessId → redirect to /business-setup

Multi-Tenancy Pattern

Every API query includes .eq("business_id", session.businessId). The Supabase service role key is used server-side only, bypassing RLS. Application code enforces all tenancy boundaries - no user can see another business's data.

16Frequently Asked Questions

Product FAQ

Q: Can a contractor use Clear Build USA without a business registered?

A: No. After email verification, new users are redirected to /business-setup and must create a business before accessing any app features. This is enforced in middleware.

Q: Can one user belong to multiple businesses?

A: Yes. The business_members table supports many-to-many. The active business is in the JWT. The top bar business switcher lets users switch context without logging out.

Q: What happens when a trial expires?

A: The business enters restricted_mode. Create flows show upgrade prompts. Existing data remains accessible in read-only mode. No data is deleted.

Q: Are customer-facing quote/invoice pages secure?

A: Quote preview pages are public (by design - customers don't have accounts). They are read-only except for the approval action, which is protected: only quotes in "sent" status can be approved, preventing replay or double-approval attacks.

Q: How does online invoice payment work?

A: The contractor sends the invoice to the customer. The customer clicks "Pay $X" on the invoice page → Stripe Checkout is created → customer pays securely on Stripe's hosted page → Stripe sends a webhook → the invoice is marked as paid automatically.

Q: Is there a mobile app?

A: The web app is fully responsive and works on mobile browsers. A dedicated React Native mobile app is on the Q3 2026 roadmap.

Q: How is my data isolated from other businesses?

A: Every database query is filtered by the businessId in your signed JWT. This is enforced server-side on every API call - no user-supplied business ID is trusted. Cross-business data access is architecturally impossible.

Q: What email provider is used?

A: Resend (resend.com). Production emails are sent from a configured domain. In local development without a Resend key, emails are logged to the console so development works without credentials.

Q: Where is data stored?

A: PostgreSQL database on Supabase (US region). File uploads (logos, etc.) stored in Supabase Storage.

Q: What is the admin panel for?

A: The admin panel (/admin) is an internal tool for the Clear Build USA team - not for customers. It provides full visibility into all businesses, users, subscriptions, and platform analytics. It has a separate login and is not linked from the main app.
