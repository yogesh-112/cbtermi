01

Tester Onboarding — Read This First

Your Tools

  • This PDF printed or on screen — fill the Result/Status columns with a pen or annotation tool
  • Google Chrome (latest) as primary browser
  • Firefox, Safari (Mac/iPhone), Edge — for compat checks
  • Mobile phone + tablet — for device checks
  • Browser DevTools (F12) — resize window for responsive
  • Screenshot tool (Windows: Win+Shift+S / Mac: Cmd+Shift+4)
  • Folder to save bug screenshots — name them BUG-001.png etc.

Test Account Setup

  • URL: https://clearbuildusa.com
  • Register a fresh account with a real email address
  • Verify your email (check inbox for verification link)
  • Complete business setup (business name, phone, address)
  • Use this account throughout all tests
  • Do NOT use a production business's real data
  • For Stripe payment tests — use card: 4242 4242 4242 4242, any future date, any CVC

How to Record Results

  • Actual Result: Write exactly what you see (even if it passed)
  • Status: Write PASS / FAIL / SKIP / N/A
  • FAIL = anything that doesn't match Expected Result
  • SKIP = test you could not run (note why)
  • N/A = test not applicable to current config
  • If FAIL: fill a Bug Report (Section 23) immediately
  • Take a screenshot for every FAIL

Daily Reporting

  • At end of each testing session, fill the Daily Report (Section 24)
  • Share the report + screenshots with the project owner
  • Do NOT move to next module until current module is fully recorded
  • Test in module order (01 → 22) — some modules depend on data from earlier ones
  • Mark ✓ next to test ID when complete
  • Ask for clarification if any Expected Result is unclear

Status Definitions

✓ PASS — Result matches expected ✗ FAIL — Result does not match, or error occurred ◌ SKIP — Could not test (note reason) — N/A — Not applicable

Priority Definitions

P1 CRITICAL — App unusable, data loss, security P2 HIGH — Major feature broken, blocks user P3 MEDIUM — Feature partially broken, workaround exists P4 LOW — Minor cosmetic, typo, UI spacing

03

Global UI/UX Checklist — Run Once Across All Pages

Brand Colors — Verify on Every Screen

Navy#16265a — Headings, sidebar, primary text
Green#3FA66B — Primary action buttons, success
Blue#2453E4 — Links, secondary actions
Background#f6f6f3 — Page background
Border#e7e6e1 — Card borders, dividers

Typography

  • Font is Inter (clean sans-serif) on all pages
  • Page titles are large, bold, navy colored
  • Body text is readable (min 14px equivalent)
  • No text overflow or truncation on standard screens
  • Labels above form fields are visible
  • No spelling errors visible on main screens

Spacing & Layout

  • Sidebar is visible and not overlapping content
  • Cards have consistent padding (approx 20-24px)
  • Page header has title + action button aligned
  • Tables have proper column spacing (not cramped)
  • Form rows have consistent spacing between fields
  • Footer / bottom content not cut off

Hover & Focus States

  • Buttons: hover changes background color visibly
  • Sidebar links: hover shows highlight
  • Table rows: hover shows light blue/gray highlight
  • Input fields: focus shows blue border outline
  • Tab key navigates through all form fields in order
  • Links: cursor changes to pointer on hover

Loading & Transitions

  • Loading spinner appears when fetching data
  • Buttons show loading state (disabled, spinner) on submit
  • Page transitions are smooth (not jarring flashes)
  • Modals open with fade animation
  • Toast notifications appear and auto-dismiss
  • No white flash on page load

Empty States

  • New account: each module shows "No X yet" message
  • Empty states have an icon + message + call-to-action button
  • No blank white boxes or missing content placeholders
  • Empty tables don't show broken headers with no rows
  • Dashboard stats show "0" not blank when no data exists

Error States

  • Form validation: red error text appears below each invalid field
  • Required fields clearly marked (asterisk or "Required")
  • API errors show user-friendly toast message (not raw JSON)
  • 404 page shows "Page not found" with a back button
  • Network timeout: app shows error, doesn't freeze
